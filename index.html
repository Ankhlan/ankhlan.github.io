Journal

A simple, classic layout — GitHub Pages ready

Once upon a time there was a person who decided to write things in a code editor — not just code but notes and journals. Like a madman with a serif font and his code set to a matching old‑style face.

The design above is intentionally restrained: narrow measure, generous leading and margins, a warm paper tone and a drop cap to evoke an old book. The type is an elegant Garamond revival that reads well on screens and prints.

Content first, ornament sparingly — the classic recipe for readable pages.

Example inline snippet() and a slightly emphasized variable are shown in the same serif family so code visually blends with the journal aesthetic.

Manual post list — edit this file to add new posts.

To add a new post (no build tools required):

  1. Create a new HTML file in the posts/ folder. Use the naming convention YYYY-MM-DD-slug.html.
  2. Edit the file title, date and content. Use the shared stylesheet at /assets/css/styles.css.
  3. Add an entry to this archive section linking to the new file (example below).
  4. Commit and push.

Example entry

<article class="post-list-item">
  <h3><a href="posts/2025-11-19-welcome.html">Welcome</a></h3>
  <time datetime="2025-11-19">2025-11-19</time>
  <p>Short excerpt for the post.</p>
</article>
