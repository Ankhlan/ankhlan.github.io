TeX islands in HTML — proof of concept

2026-04-27 metainfrastructure

This post is the first one written in the new pipeline. The architecture is a hybrid: prose lives in plain HTML so the browser can do what browsers are good at — reflow, search, accessibility, copy-paste, mobile rendering — while mathematics and figures are compiled by TeX, exported as SVG via dvisvgm, and embedded as islands inside the HTML body.

The point of the experiment is to test whether the seam between two very different layout engines holds in practice.

A simple equation

Inline math like $C_t / (1+y)^t$ is rendered by KaTeX in the browser; display math that benefits from real TeX (above) is compiled to SVG at build time and embedded. The author chooses the cost.

Why this works

The whole post lives in one file. The TeX source for each island is right where the prose references it; there is no separate figures/ folder to keep in sync, no \includegraphics{} to chase down. Editing an equation is editing the markdown; the build watcher recompiles only the islands whose source changed (cached by content hash) and reloads the page.

The web does prose well. TeX does math and diagrams well. The seam between them is an <object> tag — small, honest, and entirely under the author's control.
